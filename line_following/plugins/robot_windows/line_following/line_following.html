This simulation aims to create an algorithm so that the robot L1R2 (Lara Remote Robot) can, through sensor reading, detect and follow the line drawn on the ground, completing the route in the shortest possible time. During the journey, the robot will encounter several challenges that must be overcome.

The programming language used in the controller is Python.

The reference metric t is the elapsed time since the start of the simulation.

Time measurement will stop as soon as the robot reaches the end of the path.

Minimizing this time is one of the objectives for this simulation.

Programming a robot in Python

  1. Open the source code editor: lick on the robot with the right mouse button to pop up the robot contextual menu and select the "Edit controller" item. A window should open containing the source code of the controller of the robot.
  2. Edit the controller: a part of the code is already ready, such as the initialization of some variables, the import of libraries and the robot's initial settings. The user must implement the rest of the program.
  3. Save your program: nin the source code editor, press Ctrl-S to save your program.
  4. Reset the simulation: press the reset button to take into account your modifications.
  5. Run the simulation: press the play button to run the simulation.
  6. Observe the result: the robot should move forward. Your performance should be updated in real time on the Metrics tab. After a while, the robot should stop and your performance should be recorded (if you are logged in). Try to modify the controller program again to achieve a better performance.

During the execution, the sample controller prints some output on the console. You can open the console by clicking on the toolbar button .

L1R2

The L1R2 (Lara Remote Robot) is a line follower and rescue robot that belongs to the Laborat√≥rio Remoto em Ambiente Virtual de Aprendizagem (LARA), of the Computer Science course, at the State University of Southwest Bahia.

The robot has three light sensors at the front, three ultrasonic sensors at the front and one on each side, and three infrared sensors at the bottom.

So that the L1R2 can take the curves, it uses the concept of differential drive as locomotion mechanics. A robot is considered differentially driven when it has two motors mounted in fixed positions on its left and right side, and each of these motors is driven independently. For the robot to go forward it is necessary that both motors are moved at the same speed, and to make a turn to the left the right motor needs to be at a higher speed than the left motor and vice versa to turn to the right. It is also possible to make the robot rotate around its own axis, moving both wheels in opposite directions with the same speed. To maintain the stability of the robot, a third omnidirectional wheel is required.
